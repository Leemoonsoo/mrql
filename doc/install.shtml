MRQL: Installation Instructions

MRQL is an open-source software developed at the University of Texas at Arlington (UTA). Please read the copyright notice.

MRQL can run in two modes: in Map-Reduce mode using Apache Hadoop and in BSP mode (Bulk Synchronous Parallel mode) using Apache Hama. The BSP mode is currently experimental (not fine-tuned yet) and lacks any fault-tolerance (if an error occurs, the entire job must be restarted).

The MRQL map-reduce mode has been tested on Linux on the following Hadoop distributions:

  • Hadoop MapReduce releases 0.20.205.0, 0.21.0, 0.22.0, 1.0.1, and 1.0.3. You can download the latest hadoop-*.tar.gz from Apache.
  • Cloudera CDH3 Update 2 (cdh3u2), which includes the Hadoop MapReduce release 0.20.2 and all the latest patches. You can install the CDH3 tarball distribution from Cloudera.
The following instructions assume that you have already installed Hadoop MapReduce and you have deployed it on your cluster successfully. Otherwise, follow the directions in Running Hadoop On Ubuntu Linux. Most important change: change the JAVA_HOME in conf/hadoop-env.sh to point to your Java installation directory.

How to install MRQL

Download the latest release of MRQL (currently, version 0.8.2) and extract the files. The jar files mrql.jar and mrql-bsp.jar contain the code for MRQL map-reduce and BSP, respectively. They can be run using the scripts bin/mrql and bin/mrql.bsp.

How to run MRQL on a Hadoop MapReduce cluster:

  • Change the first lines of Makefile and bin/mrql to point to your directories. One of these directories must be your existing Hadoop configuration directory.
  • Test MRQL using bin/mrql queries/q1.mrql, which runs a simple query in map-reduce mode in memory, and using bin/mrql -local queries/q1.mrql, which runs this query in Hadoop's stand-alone mode.
  • If the existing mrql.jar doesn't match your Hadoop distribution, you can recompile MRQL using make.
For a real test, do the PageRank example on a small Hadoop MapReduce cluster.

Optional: How to run MRQL on a Hama cluster:

  • Download Apache Hama. The MRQL BSP mode has been tested on Hama 0.5.0 on top of Hadoop 1.0.3 only.
  • Follow the instructions in Getting Started with Hama to set up and start Hama.
  • Change the first lines of Makefile and bin/mrql.bsp to point to your directories. One of these directories must be your existing Hama configuration directory.
  • Test MRQL using bin/mrql.bsp queries/q1.mrql, which runs a simple query in BSP mode in memory, and using bin/mrql.bsp -local queries/q1.mrql, which runs this query in Hama's stand-alone mode.
  • If the existing mrql-bsp.jar doesn't match your Hadoop/Hama distributions, you can recompile MRQL using make bsp.
For a real test, do the PageRank example on a Hama cluster.
